💙 Guia Prático de Git & GitHub

📌 O que é o Git?

Git é um sistema de controle de versão distribuído, amplamente utilizado no desenvolvimento de software. Ele permite que várias pessoas trabalhem em um mesmo projeto ao mesmo tempo, mantendo um histórico completo de todas as alterações feitas no código.

📌 O que é o GitHub?

GitHub é uma plataforma online que funciona como um repositório remoto para projetos que usam o Git. Ele permite que você salve, compartilhe e colabore em projetos de programação com outras pessoas, mesmo que estejam em lugares diferentes do mundo.

📌 Diferenças entre Git e GitHub

  • Git: a ferramenta que gerencia versões do código no seu computador.
  • GitHub: é o serviço na nuvem que armazena e organiza repositórios Git.

📌 Principais comandos Linux para usar no Git Bash

  • pwd → mostra o diretório atual.
  • ls → lista os arquivos.
  • cd → navega entre pastas.
  • mkdir → cria uma nova pasta.
  • rm -rf → remove arquivos ou pastas.
  • clear → limpa a tela.

📌 Comandos Linux do repositório da aula

Alguns comandos encontrados no repositório:

  • git init → inicializa um repositório Git.
  • git add . → adiciona arquivos para serem versionados.
  • git commit -m "mensagem" → registra alterações no histórico.
  • git status → verifica o estado do repositório.
  • git log → mostra o histórico de commits.

📌 Como criar um repositório no GitHub

Acesse o site do GitHub Vá para github.com e entre na sua conta. Se ainda não tem uma, clique em "Sign up" para criar. Clique em “+” no canto superior direito Depois, selecione “New repository”. Preencha os dados do repositório Repository name: Escolha um nome para seu projeto. Description (opcional): Escreva uma breve descrição do que o projeto faz. Visibility: Public: Qualquer pessoa pode ver. Private: Só você (ou quem você permitir) pode ver. Marque a opção “Initialize this repository with a README” se quiser começar com um arquivo de introdução. Clique em “Create repository” Pronto! Seu repositório foi criado e está disponível online.

📌 Como ligar a um projeto existente na máquina

Dentro da pasta do projeto, use os comandos:

  • git init
  • git remote add origin URL_DO_REPOSITORIO

📌 Como gerar uma versão com Git

  • git add . → prepara os arquivos.
  • git commit -m "descrição da versão" → cria a versão.

📌 Como enviar da máquina para o GitHub

  • git push -u origin main → envia os arquivos para o repositório.
